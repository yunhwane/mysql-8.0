## 공통 코드가 필요한 이유

```sql
CREATE TABLE orders (
  order_id BIGINT PRIMARY KEY AUTO_INCREMENT,
  member_id BIGINT NOT NULL,
  order_status VARCHAR(20) NOT NULL,
  total_amount INT NOT NULL,
  created_at DATETIME NOT NULL
);

INSERT INTO orders (member_id, order_status, total_amount, created_at) VALUES
(1, '주문완료', 50000, '2026-01-15 10:30:00'),
(2, '결제완료', 75000, '2026-01-15 11:00:00'),
(3, '배송중', 30000, '2026-01-15 12:00:00'),
(1, '배송완료', 120000, '2026-01-14 09:00:00'),
(4, '주문취소', 45000, '2026-01-13 15:00:00');
```

- 주문 상태를 보자
- 개발자 A, B, C 를 봤을때 주문 상태를 다같이 잘못 입력했을때 ? 어떤 상황이 될까 ?

```sql
INSERT INTO orders (member_id, order_status, total_amount, created_at) VALUES
(5, '주문 완료', 80000, '2026-01-16 10:00:00'),
(6, 'ORDER_COMPLETE', 65000, '2026-01-16 11:00:00');
```
- 요구 사항에서 주문 완료된 상태를 찾아달라고 왔다. 그러면 어떻게 찾을 것인가?
```sql
select * from orders where order_status in ('주문완료', '주문 완료', 'ORDER_COMPLETE');
```
- 비즈니스 로직에 오류가 펼쳐질 것이다.
- 오타로 인한 버그가 생길 수 있다.
- 상태 변경이 어렵다.

### 상태 변경이 어려운 이유 ?
- 기획팀에서 '주문완료' 를 '주문성공' 을 변경요청이 온다면?
대량의 데이터 변경을 해줘야한다.
